Security Policy

Supported Versions

This project follows a single-version, rolling-release model. We only support and provide security patches for the latest released version.span_0span_0

If a vulnerability is found, it will be patched in the next release. We do not backport security fixes to older versions or specific tags.span_1span_1

Reporting a Vulnerability

Please do not open public GitHub Issues or Pull Requests for security vulnerabilities. Publicly disclosing a flaw before a fix is available puts all users of this tool at risk.span_2span_2

If you discover a security vulnerability, please report it responsibly using one of the following methods:span_3span_3

1. GitHub Private Vulnerability Reporting: Navigate to the Security tab of this repository, click on Vulnerability reporting, and select Report a vulnerability. This keeps the discussion private between you and the maintainers.span_4span_4
2. Direct Email: If private reporting is unavailable, please email the maintainer privately at Gamingwithsolex@gmail.com.

What to Include in Your Report

To help us triage and fix the issue as quickly as possible, please include:span_5span_5
• A brief description of the vulnerability and its potential impact.span_6span_6
• Step-by-step instructions to reproduce the issue (including sample code, payloads, or configuration if applicable).span_7span_7
• Any details regarding your operating system, environment, or specific installation method.span_8span_8

Our Response Process

• Acknowledgement: We will acknowledge receipt of your vulnerability report within 48 hours.span_9span_9
• Investigation: We will investigate the issue and keep you updated on our progress as we work towards a fix.span_10span_10
• Resolution: Once a fix is verified, we will release a new version containing the patch.span_11span_11
• Credit: If you wish, we will gladly credit you for the discovery in our release notes and changelog once the vulnerability has been publicly resolved.span_12span_12
